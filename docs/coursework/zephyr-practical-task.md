# Практическое задание для демонстрации языка Zephyr

## Тема задания

Разработка программы для расчета итоговых баллов студентов по набору учебных событий.

## Описание

Задание требует использовать не одну отдельную возможность языка, а несколько механизмов вместе: пользовательские типы, union types, object literals, массивы, функции, `match by`, `choose`, `collect`, модули и вывод результата.

## Цель задания

Разработать программу на языке Zephyr, которая принимает или содержит набор данных о студентах и учебных событиях, рассчитывает итоговый балл каждого студента и выводит результат в удобном виде.

Программа должна показать, что Zephyr подходит не только для простых арифметических примеров, но и для описания небольшой предметной модели с разными вариантами данных.

## Предметная область

В учебном курсе студент может получать баллы за разные события:

- экзамен;
- домашнюю работу;
- посещение;
- бонус;
- штраф.

Каждый тип события имеет собственный набор полей. Например, экзамен хранит набранный балл, максимальный балл и вес, а штраф хранит только значение, которое нужно вычесть из итоговой суммы.

Все события объединяются в общий тип `GradeEvent`. Для обработки такого значения используется discriminator-поле `kind` и конструкция `match by`.

## Требования к программе

Программа должна:

- описать типы учебных событий через `type alias`;
- объединить события в `union type`;
- описать тип студента;
- хранить для каждого студента массив учебных событий;
- рассчитывать вклад каждого события в итоговый балл;
- рассчитывать итоговый балл студента;
- переводить итоговый балл в оценку с помощью `choose`;
- формировать предупреждения с помощью `collect`;
- выводить результат в консоль;
- по возможности разделить код на несколько модулей.

## Основной алгоритм

1. Создать массив студентов.
2. Для каждого студента пройти по массиву событий.
3. Для каждого события вызвать `calculateEventScore`.
4. Сложить баллы.
5. Перевести итоговый балл в оценку через `scoreToGrade`.
6. Сформировать предупреждения.
7. Вывести имя студента, итоговый балл, оценку и предупреждения.

## Что демонстрирует задание

Задание показывает следующие возможности Zephyr:

- структурные типы;
- псевдонимы типов;
- union types;
- narrowing через `match by`;
- объектные литералы;
- массивы;
- функции;
- синтаксический сахар `choose`;
- синтаксический сахар `collect`;
- модульную организацию программы;
- вывод результата через `print`.

## Возможное усложнение

Если требуется повысить сложность задания, можно добавить:

- чтение студентов из файла через `readf`;
- запись отчета в файл через `printf`;
- сортировку студентов по итоговому баллу;
- поиск студента с максимальным баллом;
- расчет среднего балла группы;
- отдельный класс `GradeReport`;
- проверку некорректных данных, например деление на ноль при `maxScore = 0`.

## Ожидаемый результат работы программы

Программа должна вывести отчет примерно такого вида:

```text
Студент: Иван
Итоговый балл: 87
Оценка: 4
Предупреждения: нет

Студент: Мария
Итоговый балл: 94
Оценка: 5
Предупреждения: отличный результат

Студент: Олег
Итоговый балл: 52
Оценка: 2
Предупреждения: низкий итоговый балл, есть штрафы
```
