# План WebAssembly

Чтобы добавить WebAssembly в Zephyr, в существующий код нужно внести пять новых слоёв.

## 1. Lowering
- Добавить этап `lowering`, который будет понижать синтаксический сахар и сложные конструкции языка в более простую внутреннюю форму.
- Это позволит backend'ам работать не с богатым AST, а с уже нормализованным представлением.

## 2. IR
- Добавить `IR` (`Intermediate Representation`) как промежуточное представление между semantic layer и backend'ами.
- В `IR` должны быть явно выражены функции, переменные, control flow, вызовы, доступ к полям и массивам.

## 3. Wasm backend
- Добавить новый `backend/wasm`, который будет генерировать Wasm из IR.
- На первом этапе он должен покрыть базовые конструкции: арифметику, функции, locals/globals, control flow и массивы.

## 4. Runtime helper'ы
- Добавить минимальные runtime helper'ы для Wasm: память, строки, массивы и базовую работу со значениями.
- Это будет маленький runtime-слой, на который сможет опираться Wasm backend.

## 5. JS glue layer
- Добавить тонкий JS-слой вокруг `.wasm`, который будет загружать модуль, пробрасывать imports и оборачивать exports.
- Через этот слой Wasm сможет встраиваться в браузер и Node и взаимодействовать с внешним JS-кодом.
